# PLC 学习案例说明

这个项目现在不只是演示“读写寄存器”，还加入了几个更接近真实上位机的页面案例。

它们都共用同一套底层流程：

```text
选择案例
  -> 套用起始地址 / 读取数量 / 写入值
  -> 通过 IPlcClient 读取或写入寄存器
  -> MainViewModel 把寄存器翻译成业务状态
  -> WPF Binding 自动刷新不同页面
```

这和后端 API 项目里“从外部系统拿到原始数据，再组装成页面 DTO 或响应模型”是同一个思路。

## 1. 设备状态监控

用途：学习最常见的设备监控看板。

寄存器范围：

```text
HR0 - HR7
```

寄存器映射：

| 地址 | 含义 | 说明 |
| --- | --- | --- |
| HR0 | 运行状态 | 0=停机，1=运行，2=待机 |
| HR1 | 当班产量 | 件数 |
| HR2 | 温度 | 按 0.1 缩放，例如 680 表示 68.0 °C |
| HR3 | 压力 | 按 0.1 缩放，例如 58 表示 5.8 bar |
| HR4 | 电机转速 | rpm |
| HR5 | 良率 | 按 0.1 缩放，例如 986 表示 98.6% |
| HR6 | 当前节拍 | ms |
| HR7 | 报警字 | 0 表示无报警，非 0 表示存在报警位 |

推荐操作：

1. 协议选择 `Simulation`
2. 左侧案例选择“设备状态监控”
3. 点击“套用案例到读写参数”
4. 点击“连接”
5. 点击“读取保持寄存器”
6. 查看“监控总览”和“寄存器”页面

如果连续读取或启动自动刷新，也可以切到“趋势”页面。
趋势页会记录每次读取返回的第一个寄存器值，并显示最大值、最小值和最新采样时间。

学习重点：

- 一个寄存器不等于一个文本，通常需要状态码翻译
- 很多现场值会按比例缩放，例如温度、压力、良率
- 报警字通常是位域，真实项目还要继续拆 bit
- 趋势曲线通常来自周期采样，不是 PLC 自动给你一张图

## 2. 产线节拍分析

用途：学习多工序状态和节拍瓶颈展示。

寄存器范围：

```text
HR10 - HR15
```

寄存器映射：

| 地址 | 含义 | 说明 |
| --- | --- | --- |
| HR10 | 上料状态 | 0=待机，1=运行中，2=完成，3=故障 |
| HR11 | 加工状态 | 0=待机，1=运行中，2=完成，3=故障 |
| HR12 | 检测状态 | 0=待机，1=运行中，2=完成，3=故障 |
| HR13 | 下料状态 | 0=待机，1=运行中，2=完成，3=故障 |
| HR14 | 当前节拍 | ms |
| HR15 | 瓶颈提示 | 0=无瓶颈，1=存在瓶颈 |

推荐操作：

1. 左侧案例选择“产线节拍分析”
2. 点击“套用案例到读写参数”
3. 读取 HR10-HR15
4. 查看“产线工序”页面

如果启动自动刷新，可以再看“趋势”页面观察当前节拍或首个工序状态的变化。

学习重点：

- 多个寄存器组合成一个业务页面
- 状态码需要翻译成业务文案
- 节拍和瓶颈通常来自 PLC 内部统计或上位机二次计算

## 3. 安全联锁排查

用途：学习设备启动前的联锁条件判断。

寄存器范围：

```text
HR20 - HR25
```

寄存器映射：

| 地址 | 含义 | 期望值 | 说明 |
| --- | --- | --- | --- |
| HR20 | 安全门闭合 | 1 | 安全门未闭合时禁止启动 |
| HR21 | 急停释放 | 0 | 急停按下时禁止动作 |
| HR22 | 光栅无遮挡 | 1 | 光栅被遮挡时禁止动作 |
| HR23 | 气压正常 | 1 | 气压不足会导致气缸不到位 |
| HR24 | 伺服就绪 | 1 | 伺服未就绪不能自动运行 |
| HR25 | 无复位请求 | 0 | 复位请求存在时先执行复位 |

推荐操作：

1. 左侧案例选择“安全联锁排查”
2. 点击“套用案例到读写参数”
3. 读取 HR20-HR25
4. 查看“安全联锁”和“报警”页面

学习重点：

- 联锁不是单个值，而是一组条件同时满足
- 页面应该显示实际值、期望值和判断结果
- 自动刷新时要避免同一报警无限刷屏，所以项目里做了轻量去重

## 4. 参数写入验证

用途：学习参数下发后读回验证。

寄存器范围：

```text
HR30 - HR32
```

寄存器映射：

| 地址 | 含义 | 说明 |
| --- | --- | --- |
| HR30 | 温度设定值 | 按 0.1 缩放 |
| HR31 | 温度实际值 | 按 0.1 缩放 |
| HR32 | 允许偏差 | 按 0.1 缩放 |

推荐操作：

1. 左侧案例选择“参数写入验证”
2. 点击“套用案例到读写参数”
3. 点击“写入单个寄存器”，把 HR30 写成建议值
4. 点击“读取保持寄存器”
5. 查看“参数验证”页面

学习重点：

- 写入成功不代表设备一定生效
- 工业现场常见做法是写入后再读回
- 读回后还要和实际值、允许偏差一起判断

## 6. 后续可以继续扩展的案例

如果继续加案例，我建议优先做这些：

1. 电机启停控制：启动、停止、正转、反转、故障复位
2. 模拟量采集：温度、压力、液位、电流、电压
3. 配方管理：多个参数组合下发并校验
4. 报警位解析：把一个报警字拆成 16 个 bit
5. 设备 OEE：开机率、性能、良率的综合计算
